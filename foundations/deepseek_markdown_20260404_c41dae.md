## 從物理直覺到 150+ 種程式語言生態系的旅程

一切始於一堂七年級的物理課。老師以公車行駛途中緊急煞車，導致乘客向前傾倒為例，來說明「慣性」的概念。

老師解釋：「當車輛行駛時，乘客與車輛具有相同速度。當車輛煞車，輪胎與路面的摩擦力產生制動力使車輛減速停止。然而，由於慣性，乘客的身體傾向於保持原本的運動狀態，因此繼續向前移動。」當時，一個 13 歲男孩的腦中浮現了一個疑問：*那股無形推動我們的力量究竟是什麼？它有多強？我們能否在車輛煞車前就計算出它？*

我前往市立圖書館，翻遍了館內書架，卻找不到任何一本書有系統地、定量地定義慣性。科學界似乎將慣性視為一種「顯而易見的事實」，而非一個可計算的變數。

懷抱著解開名為「慣性」的無形力量之抱負，男孩報名了高中物理資優班的補習課程，期望能考上資優班，進而就讀大學物理系，屆時便能找到解答，揭開疑惑，將「慣性」公諸於世。

歷經兩年勤奮準備，男孩迎來了關鍵的考試。他根據自己從圖書館書籍中理解的方式作答，而非遵循老師教導的方法。學校公布結果那天，天旋地轉，手中握著落榜通知，眼淚奪眶而出，心臟緊縮、胸悶到幾乎無法呼吸。就這樣，失去了找出無形力量「慣性」的機會。

無法承受如此巨大的傷痛，男孩休學一年，不再上學，即便父母打罵、威脅將他逐出家門也無濟於事。他孤獨一人，無人理解。他反覆自問：我錯在哪裡？我並不懶惰，也不愚鈍。國中四年，我的物理成績始終名列班上前五名。為何我會落榜？為何學校不錄取我？他卑微地希望，那所資優班能允許他站在窗外，聆聽老師們講授物理就好。

在痛苦與折磨中，一年過去了。新學年開始，他就讀於市內一所普通高中。未能進入夢想中的物理資優班，雖然關閉了接受菁英訓練的大門，卻也開啟了一條強烈的自學之路。我完成高中學業，考上了父母期望的財務會計系，但我仍持續自學物理，追尋真理。

大學就讀會計系的歲月，給了我一個截然不同的視角：**數字的嚴謹性**。在會計學中，每一筆交易都必須根據不變的原則進行記帳與平衡。我心想：*如果經濟活動可以如此嚴謹地數位化，為何我們不將宇宙法則也數位化，讓電腦能夠絕對精確地執行它們？* 這正是 **NKTg 定律** 誕生的第一顆種子。

許多人問我：「為何要用超過 150 種程式語言實作 NKTg？這是在作秀嗎？」

答案是：**不是**。

將 NKTg 帶入龐大的語言生態系——從貼近硬體的底層語言到現代高階語言——是為了證明一個真理：**自然定律是永恆且一致的**。無論你用 Python、C++、Assembly 還是 SQL 來撰寫，變動慣性的本質都不會被扭曲。因為物理的真實只有一個，所以輸出結果必須相同。

另一方面，每種程式語言的誕生都有其獨特的使命，並適合特定程式設計師的思維模式。我希望 NKTg 能存在於數位世界的每個角落，讓任何人、在任何領域，都能使用他們所擅長的「語言」與宇宙對話。

## 核心需求：為何 AI 時代需要具備可計算性的物理定律？

我們並非在此反駁經典物理定律。相反地，**NKTg 演算法** 的誕生，源於觀察舊理論在面對數位化現實時的「斷裂點」。

在牛頓或愛因斯坦的物理學中，質量 (m) 在基本動力學問題中通常被視為固定常數。但請看看機器人時代的現實：一架無人機 (Drone) 在移動、接收和投遞貨物的過程中，其質量 (dm/dt) 在每一瞬間都在變化。當 m 不再是常數時，基於舊有基礎的演算法便開始顯現誤差。

這就是為何世界需要一個具備「可計算性」的定律，它具有以下幾項優勢：

- **與機器學習/AI 的絕對相容性：** NKTg 不要求 AI 去「學習」那些繁複的微分方程式。它提供了一個直接的函數，將變動的質量視為輸入參數，而非預設的常數。

- **即時反應能力：** 在控制程式設計中，每一毫秒都至關重要。NKTg 直接進行計算，而非耗費 CPU/GPU 資源去解複雜的方程組。這使得系統反應更快、更流暢，無需沉重的模擬器。

- **「數位化」思維與風險管理：** 繼承了會計領域的嚴謹思維，NKTg 演算法將變數 x, m, v, dm/dt 視為一個記帳系統的輸入數據。回傳的結果不只是一串數字，而是**運動趨勢**。這使得 AI 和機器人能夠在穩定系統過載風險發生前進行預測和行為調整。

- **數據完整性：** NKTg 的運作完全基於來自感測器和測量設備的真實數據。我們摒棄了假設的常數、近似計算或手動誤差補償係數。在 NKTg 中，真實數據就是最終的答案。

# 第一章：核心函數 NKTg_ENGINE 的結構 (x, v, m)

在本章中，我們將把物理觀察轉化為可執行的演算法結構。NKTg 演算法不處理靜態的微分方程，而是專注於透過動態數據對來確定**運動趨勢**。

## 1. 數學邏輯與趨勢分類

該演算法基於動量 (p = m \cdot v)，透過兩個基本的乘積量來確定系統狀態：

- **NKTg1 = x \* p (軌道穩定性參數)：** 位置定向。

  - 若 **NKTg1 > 0**：物體趨向於遠離穩定狀態。
  - 若 **NKTg1 < 0**：物體趨向於回到穩定狀態。
  - 若 **|NKTg1| < 誤差閾值 (零區)**：系統達到穩定平衡。

- **NKTg2 = (dm/dt) \* p (質量-動量交換參數)：** 確定質量變化的影響。
  - 若 **NKTg2 > 0**：質量變化正在輔助運動。
  - 若 **NKTg2 < 0**：質量變化正在抵抗運動（制動作用）。

## 2. 輸入參數定義

為達到最高實驗精度 (**99.792%**)，對應於航空航天問題中僅 **0.208%** 的誤差（與 ESA 數據比對），演算法優先使用 64 位元浮點數資料類型 (**float64/double**)。

- **x (float64)：** 視問題而定，x 可以是到中心的絕對距離（天文學），或是相對於期望平衡位置的偏移量（控制工程）。

- **v (float64)：** 瞬時速度 (m/s)。

- **m (float64)：** 瞬時質量 (kg)。

- **dm_dt (float64)：** 質量變化率 (kg/s)。在實務中，此值從實驗測量中提取，或透過微分計算：(m_{新} - m_{舊}) / \Delta t。

## 3. 函數執行結構與抗噪邏輯 (零區)

在實際環境中，感測器數據總是帶有雜訊。NKTg 演算法整合了一個**零區**緩衝區，以避免當數值在零附近震盪時，控制系統持續抖動。

**演算法虛擬碼：**

```python
# 設定誤差閾值以使系統穩定
ZERO_ZONE = 0.001

FUNCTION NKTg_Engine(x, v, m, dm_dt):
    # 步驟 1：計算動量 p
    p = m * v

    # 步驟 2：計算乘積指標
    nktg1 = x * p
    nktg2 = dm_dt * p

    # 步驟 3：使用零區進行趨勢分類
    IF absolute_value(nktg1) < ZERO_ZONE:
        tendency1 = "Stable equilibrium"
    ELSE IF nktg1 > 0:
        tendency1 = "Moving away from stable state"
    ELSE:
        tendency1 = "Moving toward stable state"

    IF absolute_value(nktg2) < ZERO_ZONE:
        tendency2 = "No mass variation effect"
    ELSE IF nktg2 > 0:
        tendency2 = "Mass variation supports movement"
    ELSE:
        tendency2 = "Mass variation resists movement"

    RETURN {p, nktg1, nktg2, tendency1, tendency2}
